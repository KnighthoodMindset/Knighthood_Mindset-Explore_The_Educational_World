Ping Sweep (ICMP Scanning)** #### **Definition:** Ping Sweep is a technique used to determine **which IP addresses are active** or "alive" on a network. It sends **ICMP Echo Requests** to multiple IP addresses and waits for **ICMP Echo Replies**. #### **How It Works:** * Sends a **ping (ICMP Echo Request)** to each IP address. * Waits for a **response (ICMP Echo Reply)**. * Hosts that respond are marked as **live**. #### **Purpose:** * Identify live systems in a subnet. * Eliminate non-responsive IPs from further scanning. #### **Limitations:** * ICMP may be blocked by firewalls. * Can trigger intrusion detection systems (IDS). #### **Example Tool:** `fping`, `nmap`, `ping` --- ### üåê **2. Port Scanning** #### **Definition:** Port scanning is used to discover **open ports** and **services** on a target system. Each port corresponds to a particular service or application. #### **Types of Port Scanning:** | Technique | Description | | ----------------------------- | --------------------------------------------------------------------------------------------------------------- | | **TCP Connect Scan** | Establishes full TCP connection (3-way handshake). Reliable but noisy. | | **SYN Scan (Half-open scan)** | Sends only SYN packets. If SYN-ACK is received, port is open. Stealthier. | | **FIN Scan** | Sends FIN packets without connection. If no response, the port may be open. Used to evade firewalls. | | **NULL Scan** | Sends a packet with no flags set. Open ports remain silent. Can bypass stateless firewalls. | | **XMAS Scan** | Sends a packet with FIN, URG, and PUSH flags set (like a Christmas tree). Responses are interpreted for status. | #### **Purpose:** * Identify open/closed/filtered ports. * Discover services and daemons. * Map possible attack entry points. #### **Example Tool:** `nmap`, `Netcat` --- ### üìç **3. Network Scanning** #### **Definition:** Network scanning is used to **identify active devices**, **network topology**, and **potential vulnerabilities** in a network. #### **Techniques:** * ARP scanning for local networks * ICMP scanning for remote hosts * TCP/UDP port detection * Banner grabbing for identifying services #### **Purpose:** * Map network devices * Identify unauthorized systems * Determine available services #### **Example Tools:** `Nmap`, `Angry IP Scanner` --- ### üß± **4.
