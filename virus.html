Lifecycle and Classification of Virus

1. Introduction

A computer virus is a type of malicious software designed to replicate itself and spread from one computer to another, often causing damage or disrupting normal system operations. Understanding the virus lifecycle is crucial for ethical hackers and cybersecurity professionals to detect, analyze, and prevent virus infections effectively.

Additionally, viruses come in many types, each with distinct characteristics, infection methods, and payloads. Classifying viruses helps in identifying their behavior and developing appropriate countermeasures.

2. Lifecycle of a Virus

The lifecycle of a virus describes the sequential stages it undergoes from initial infection to spreading and causing harm. Although the exact details may vary depending on the virus type, the lifecycle generally includes the following phases:

a. Penetration (Entry)
  • The virus gains access to the host system.
  • Entry vectors include:
    • Executing infected files.
    • Opening infected email attachments.
    • Using infected removable media (USB drives).
    • Exploiting network vulnerabilities.
  • At this stage, the virus is typically dormant and not yet active.
b. Activation (Triggering)
  • The virus waits for a specific condition or trigger to activate.
  • Triggers can be:
    • A specific date/time (e.g., April 1st).
    • Execution of a particular program.
    • System startup.
    • User actions.
  • This delayed activation helps evade immediate detection.
c. Replication
  • The virus replicates itself by creating copies.
  • Replication methods:
    • Infecting other executable files (e.g., .exe, .com).
    • Infecting boot sectors of storage devices.
    • Injecting code into system processes or memory.
  • This enables the virus to spread within the host system and potentially to other systems.
d. Execution of Payload
  • The virus executes its payload, which is the malicious part.
  • Payload actions can vary widely:
    • Deleting or corrupting files.
    • Logging keystrokes.
    • Opening backdoors for remote access.
    • Displaying messages or prank behavior.
    • Consuming system resources causing slowdowns.
  • Some viruses have no payload, their primary purpose being replication.
e. Concealment and Evasion
  • To avoid detection by antivirus software, viruses employ evasion techniques:
    • Polymorphism: Virus changes its code each time it replicates.
    • Metamorphism: Virus rewrites its own code entirely.
    • Encryption: Virus encrypts its code and decrypts at runtime.
    • Rootkit techniques: Hides processes and files from the OS.
  • These techniques make detection and removal difficult.
f. Propagation to Other Systems
  • After infecting the host system, the virus attempts to spread to other computers.
  • Methods include:
    • Infecting files shared via network shares.
    • Emailing infected attachments.
    • Infecting removable media.
    • Exploiting network vulnerabilities.

3. Classification of Viruses

Viruses are classified based on various factors including infection method, target location, and behavior. Below are the common classifications:

a. File Infectors
  • Infect executable files (.exe, .com).
  • Activate when the infected file is run.
  • Example: Cascade virus.
b. Boot Sector Viruses
  • Infect the boot sector of hard drives or removable disks.
  • Activate during system startup before the OS loads.
  • Can cause system failure and make detection difficult.
  • Example: Michelangelo virus.
c. Macro Viruses
  • Infect documents that contain macros (e.g., Microsoft Word, Excel).
  • Triggered when the document is opened and macros execute.
  • Often used in phishing campaigns.
  • Example: Melissa virus.
d. Polymorphic Viruses
  • Change their code structure each time they infect to avoid detection.
  • Use encryption and code obfuscation.
  • Example: Marburg virus.
e. Metamorphic Viruses
  • Rewrite their entire code with each infection.
  • More complex than polymorphic viruses, harder to detect.
  • Example: W32/Simile virus.
f. Resident Viruses
  • Embed themselves into system memory.
  • Can infect files and processes regardless of which program is running.
  • Stay active until system reboot.
  • Example: Randex virus.
g. Non-resident Viruses
  • Infect files and then terminate.
  • They do not stay resident in memory.
  • Example: Vienna virus.
h. Stealth Viruses
  • Hide their presence by intercepting system calls.
  • Provide fake clean results to antivirus scans.
  • Example: Brain virus.
i. Multipartite Viruses
  • Infect multiple parts of the system such as files and boot sectors.
  • More difficult to remove as they have multiple infection points.
  • Example: One_Half virus.

5. Conclusion

The lifecycle of a virus highlights the phases from initial infection to spreading and payload execution, while the classification of viruses helps identify their unique characteristics and infection methods. For ethical hackers, this knowledge is critical to detect infections early, analyze virus behavior, and develop effective mitigation strategies.
