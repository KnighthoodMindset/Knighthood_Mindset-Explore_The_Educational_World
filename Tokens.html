Tokens

In compiler design, a token is the smallest unit of a program that has meaning to the compiler. Tokens are produced by the lexical analyzer (scanner) during the first phase of compilation and are passed to the syntax analyzer for further processing.

Tokens represent the building blocks of source code, such as keywords, identifiers, operators, constants, and symbols.

A token is a pair:

  • Token-name: Category of the token (e.g., 'IDENTIFIER', 'NUMBER', 'PLUS').
  • Attribute-value: Extra information needed to understand the token (e.g., the name of a variable, the actual number).
    • Token Type Description Example
    Keywords Reserved words with special meaning in a language if, while, int
    Identifiers Names given to variables, functions, arrays, etc. sum, count
    Constants Literal values like integers, floating-point numbers, chars 10, 'a', 3.14
    Operators Symbols that perform operations +, -, *, /
    Separators Punctuation symbols used to separate code elements ,, ;, {, }
    Strings Sequence of characters enclosed in double quotes "Hello"
    Comments Non-executable notes in code, ignored by the compiler // comment

    Example Code and Corresponding Tokens

    int x = 5 + y;

    Code Element Token
    int <KEYWORD, int>
    x <IDENTIFIER, x>
    = <OPERATOR, =>
    5 <CONSTANT, 5>
    + <OPERATOR, +>
    y <IDENTIFIER, y>
    ; <SEPARATOR, ;>

    Role of Tokens in Compilation

  • Tokens are the input for the syntax analyzer(parser).
  • They simplify the grammar by reducing the number of raw character rules.
  • They represent lexemes that conform to defined patterns in the language.

    • Token vs Lexeme vs Pattern

  • Token: Symbol that represents a set of strings (e.g., 'IDENTIFIER')
  • Lexeme: Actual string in the source code that matches the pattern (e.g., 'x')
  • Pattern: Rule describing what a lexeme should look like (e.g., '[a-zA-Z]+')

    • Conclusion

    Tokens are the basic symbols recognized by the lexical analyzer, and they serve as the foundation of the source program’s structure. They help in efficient parsing and semantic analysis and are essential to bridge human-readable code and machine-level understanding.
