Installing and Testing Damn Vulnerable Web Application (DVWA)

Introduction

Damn Vulnerable Web Application (DVWA) is designed to be a hands-on platform for learning web application security. To effectively use DVWA, one must first install it in a safe environment and then proceed to test its vulnerabilities.

This section covers step-by-step instructions on installing DVWA and guidelines on how to begin testing its vulnerabilities, ensuring a smooth learning experience for beginners and professionals alike.

Prerequisites for Installing DVWA

Before installing DVWA, ensure you have the following software components:

  1. Web Server: Apache HTTP Server (commonly part of XAMPP or LAMP stacks)
  2. Database Server: MySQL or MariaDB to store DVWA data
  3. PHP: Version 5.6 or higher, with extensions like mysqli, gd, and mbstring
  4. Operating System: DVWA can be installed on Windows, Linux, or macOS platforms.

For convenience, many users prefer all-in-one packages such as:

  • XAMPP (Cross-platform Apache, MariaDB, PHP, Perl) for Windows, Linux, macOS
  • Bitnami DVWA Stack which bundles everything pre-configured

Step-by-Step Installation Guide

Step 1: Download DVWA
  • Visit the official DVWA GitHub repository: https://github.com/digininja/DVWA
  • Download the latest release as a ZIP file or clone the repository using Git:
git clone https://github.com/digininja/DVWA.git
Step 2: Set Up Your Web Server Environment
  • If not already installed, download and install XAMPP or similar.
  • Start Apache and MySQL services from the control panel.
Step 3: Deploy DVWA Files
  • Extract the downloaded DVWA files into your web server’s root directory:
    • For XAMPP on Windows, typically: C:\xampp\htdocs\dvwa\
    • For Linux, it may be /var/www/html/dvwa/
Step 4: Configure DVWA
  • Navigate to the DVWA directory and find the file config/config.inc.php.dist
  • Rename it to config.inc.php
  • Open config.inc.php in a text editor and configure database credentials:
$_DVWA = array();
$_DVWA[ 'db_server' ]   = '127.0.0.1';      // usually localhost or 127.0.0.1
$_DVWA[ 'db_database' ] = 'dvwa';
$_DVWA[ 'db_user' ]     = 'root';            // default MySQL username
$_DVWA[ 'db_password' ] = '';                // default MySQL password, blank for XAMPP

Make sure the credentials match your MySQL setup.

Step 5: Create DVWA Database
  • Access phpMyAdmin via your web browser (usually at http://localhost/phpmyadmin/).
  • Create a new database named dvwa.
  • Alternatively, run the following MySQL commands:
CREATE DATABASE dvwa;
GRANT ALL PRIVILEGES ON dvwa.* TO 'root'@'localhost' IDENTIFIED BY '';
FLUSH PRIVILEGES;

Adjust username and password as needed.

Step 6: Initialize DVWA Database
  • Open your browser and go to: http://localhost/dvwa/setup.php
  • Click on Create / Reset Database.
  • You should see a confirmation message indicating the database has been initialized.

Testing DVWA Vulnerabilities

Once DVWA is installed and configured, you can begin testing its vulnerabilities:

  1. Log in with default credentials:
    • Username: admin
    • Password: password
  2. Set Security Level:
    • DVWA allows you to choose security levels: Low, Medium, High, or Impossible.
    • Start with Low to understand basic vulnerabilities.
    • Increase levels to learn about protective measures and hardened systems.
  3. Explore Vulnerabilities:
    • Navigate to modules such as SQL Injection, XSS, Command Injection, etc.
    • Try exploiting vulnerabilities by entering crafted inputs and observing the behavior.
    • Understand how attacks work and how DVWA responds.
  4. Apply Fixes:
    • Modify DVWA source code to patch vulnerabilities.
    • Test how fixes prevent exploitation.
    • This practice helps in learning secure coding and remediation.

Safety Considerations

  • Run DVWA only on isolated machines or virtual environments.
  • Never expose DVWA to the internet or public networks.
  • Use strong passwords and limit access to your lab setup.

DVWA is intentionally insecure, so maintaining strict containment prevents accidental abuse or hacking of other systems.

Conclusion

Installing and testing DVWA is a foundational exercise for anyone pursuing ethical hacking and web application security. By following the installation steps and actively engaging with DVWA’s vulnerabilities, learners gain crucial practical insights into common security flaws and effective defense mechanisms. This hands-on experience builds confidence and competence in identifying and mitigating risks in real-world applications.
